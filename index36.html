Sparkroot — Supabase Setup
Almost ready! Complete these 4 steps.

Supabase is the free, easy alternative to Firebase. No credit card needed. Takes about 10 minutes.

1

Create a free Supabase account & project

Go to supabase.com, click "Start your project", and sign in with GitHub or email. Then click "New project", name it sparkroot, set any database password, pick the closest region, and click Create new project. Wait about 1 minute for it to set up.

2

Copy your Project URL & anon key

In your project dashboard, click Settings (gear icon) → API. You'll see two things to copy:

• Project URL — looks like https://abcdef.supabase.co
• anon public key — a long string starting with eyJ...

Open this HTML file in a text editor, find the section marked:
// ▼▼▼ PASTE YOUR SUPABASE CONFIG HERE ▼▼▼
and replace the two placeholder values with your real ones.

3

Create your database tables

In your Supabase project, click SQL Editor in the left sidebar, then click New query. Paste the entire block below and click Run:

create table profiles (id uuid references auth.users primary key, name text not null, email text, role text not null default 'little', bio text, resume_url text, school text, interests text[] default '{}', colleges_looking text[] default '{}', career_fields text[] default '{}', education text, company text, location text, willing_to text[] default '{}', created_at timestamptz default now()); create table posts (id uuid default gen_random_uuid() primary key, author_id uuid, author_name text, author_role text, subtitle text, tag text default 'general', text text, liked_by text[] default '{}', helpful_by text[] default '{}', replies jsonb default '[]', created_at timestamptz default now()); create table career_questions (id uuid default gen_random_uuid() primary key, career_key text, question text, asked_by text, asked_by_id uuid, answer text, created_at timestamptz default now()); create table college_reviews (id uuid default gen_random_uuid() primary key, college_id text, author_id uuid, author_name text, stars numeric default 5, safety_stars integer default 5, support_stars integer default 5, inclusivity_stars integer default 5, social_stars integer default 5, academics_stars integer default 5, text text, created_at timestamptz default now()); create table college_questions (id uuid default gen_random_uuid() primary key, college_id text, question text, asked_by text, asked_by_id uuid, answer text, created_at timestamptz default now()); create table notifications (id uuid default gen_random_uuid() primary key, user_id uuid not null, type text, message text, read boolean default false, created_at timestamptz default now()); alter table profiles enable row level security; alter table posts enable row level security; alter table career_questions enable row level security; alter table college_reviews enable row level security; alter table college_questions enable row level security; alter table notifications enable row level security; create policy "Public read profiles" on profiles for select using (true); create policy "Own insert profile" on profiles for insert with check (auth.uid() = id); create policy "Own update profile" on profiles for update using (auth.uid() = id); create policy "Public read posts" on posts for select using (true); create policy "Auth insert post" on posts for insert with check (auth.role() = 'authenticated'); create policy "Auth update post" on posts for update using (auth.role() = 'authenticated'); create policy "Public read career_questions" on career_questions for select using (true); create policy "Auth insert career_question" on career_questions for insert with check (auth.role() = 'authenticated'); create policy "Auth update career_question" on career_questions for update using (auth.role() = 'authenticated'); create policy "Public read college_reviews" on college_reviews for select using (true); create policy "Auth insert college_review" on college_reviews for insert with check (auth.role() = 'authenticated'); create policy "Public read college_questions" on college_questions for select using (true); create policy "Auth insert college_question" on college_questions for insert with check (auth.role() = 'authenticated'); create policy "Own read notifications" on notifications for select using (auth.uid() = user_id); create policy "Auth insert notification" on notifications for insert with check (auth.role() = 'authenticated'); create policy "Own update notification" on notifications for update using (auth.uid() = user_id); alter publication supabase_realtime add table posts;
⚠️ You only need to run this SQL once. If you see errors about tables already existing, that's fine — just ignore them.

Already set up previously? Run this extra SQL to add new features (profiles, notifications, college ratings):
-- College review category columns alter table college_reviews add column if not exists safety_stars integer default 5; alter table college_reviews add column if not exists support_stars integer default 5; alter table college_reviews add column if not exists inclusivity_stars integer default 5; alter table college_reviews add column if not exists social_stars integer default 5; alter table college_reviews add column if not exists academics_stars integer default 5; alter table college_reviews alter column stars type numeric; create policy "Auth update career_question" on career_questions for update using (auth.role() = 'authenticated'); -- Profile extended fields alter table profiles add column if not exists bio text; alter table profiles add column if not exists resume_url text; alter table profiles add column if not exists school text; alter table profiles add column if not exists interests text[] default '{}'; alter table profiles add column if not exists colleges_looking text[] default '{}'; alter table profiles add column if not exists career_fields text[] default '{}'; alter table profiles add column if not exists education text; alter table profiles add column if not exists company text; alter table profiles add column if not exists location text; alter table profiles add column if not exists willing_to text[] default '{}'; -- Notifications table create table if not exists notifications (id uuid default gen_random_uuid() primary key, user_id uuid not null, type text, message text, read boolean default false, created_at timestamptz default now()); alter table notifications enable row level security; create policy "Own read notifications" on notifications for select using (auth.uid() = user_id); create policy "Auth insert notification" on notifications for insert with check (auth.role() = 'authenticated'); create policy "Own update notification" on notifications for update using (auth.uid() = user_id);
4

Save and deploy to Netlify

Once your config is pasted and saved, go to netlify.com/drop, drag and drop this sparkroot.html file, and you're live! Your site will be at a URL like sparkroot-abc123.netlify.app.

⚙️ Admin Setup SQL

Run this once in Supabase → SQL Editor to enable the admin panel's delete permissions and custom colleges/career fields tables.

-- 1. Add admin flag to profiles alter table profiles add column if not exists is_admin boolean default false; -- 2. Set yourself as admin (run AFTER your profile exists) update profiles set is_admin = true where email = 'rylakrulik@gmail.com'; -- 3. Allow admin to delete any post do $$ begin create policy "Admin delete post" on posts for delete using ( exists (select 1 from profiles where id = auth.uid() and is_admin = true)); exception when duplicate_object then null; end $$; -- 4. Allow admin to delete any profile do $$ begin create policy "Admin delete profile" on profiles for delete using ( exists (select 1 from profiles where id = auth.uid() and is_admin = true)); exception when duplicate_object then null; end $$; -- 5. Custom colleges table create table if not exists admin_colleges ( id text primary key, name text not null, location text default '', emoji text default '🎓', color text default '#833ab4', tags text[] default '{}', created_at timestamptz default now()); alter table admin_colleges enable row level security; do $$ begin create policy "Public read admin_colleges" on admin_colleges for select using (true); exception when duplicate_object then null; end $$; do $$ begin create policy "Admin manage admin_colleges" on admin_colleges for all using ( exists (select 1 from profiles where id = auth.uid() and is_admin = true)); exception when duplicate_object then null; end $$; -- 6. Custom career fields table create table if not exists admin_career_fields ( key text primary key, emoji text default '💼', title text not null, created_at timestamptz default now()); alter table admin_career_fields enable row level security; do $$ begin create policy "Public read admin_career_fields" on admin_career_fields for select using (true); exception when duplicate_object then null; end $$; do $$ begin create policy "Admin manage admin_career_fields" on admin_career_fields for all using ( exists (select 1 from profiles where id = auth.uid() and is_admin = true)); exception when duplicate_object then null; end $$;

After running, log out and back in — the ⚙️ Admin button will appear in the top nav.
